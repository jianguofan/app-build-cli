# End-to-End Testing Guide

## 测试概述

本文档描述 App Build Platform 的端到端测试流程，确保所有核心功能正常工作。

---

## 测试环境准备

### 前置条件

1. **宿主机环境**:
   - macOS 系统
   - Xcode 已安装（iOS 构建）
   - Android Studio + Android SDK 已安装
   - Flutter SDK 已安装
   - Docker Desktop 已安装并运行

2. **SSH 配置**:
   ```bash
   # 启用远程登录
   sudo systemsetup -setremotelogin on
   
   # 生成 SSH 密钥（如果没有）
   ssh-keygen -t rsa -b 4096
   
   # 添加公钥到 authorized_keys
   cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
   
   # 测试 SSH 连接
   ssh localhost
   ```

3. **工作空间目录**:
   ```bash
   mkdir -p ~/app-build-workspace/{projects,builds/{ios,android},logs}
   ```

4. **环境变量配置**:
   - 复制 `.env.example` 到 `.env`
   - 填写所有必需的 API 密钥和配置

## 已知问题和限制

### Phase 1 限制

1. **单队列执行**: 
   - 构建任务按顺序执行，不支持并发
   - 影响: 多个任务需要排队等待

2. **内存存储**:
   - 数据存储在内存中
   - 影响: 服务重启后数据丢失

3. **硬编码认证**:
   - 只有一个管理员账号
   - 影响: 无法区分不同用户的操作

4. **模拟发布**:
   - 部分发布器使用模拟上传（Phase 1）
   - 影响: 实际上传需要在 Phase 2 实现

5. **无构建缓存**:
   - 每次构建都是完整构建
   - 影响: 构建时间较长

### 常见问题

1. **SSH 连接失败**:
   - 原因: SSH 密钥配置不正确
   - 解决: 检查 SSH_USER 和 SSH_KEY_PATH 环境变量

2. **构建超时**:
   - 原因: 构建时间超过 30 分钟
   - 解决: 检查网络连接和依赖下载速度

3. **WebSocket 断开**:
   - 原因: 网络不稳定或代理问题
   - 解决: 刷新页面重新连接

4. **发布失败**:
   - 原因: API 密钥配置错误
   - 解决: 检查 .env 文件中的配置
